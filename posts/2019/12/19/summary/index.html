/x00

/x00截断

0x00是一种十六进制表示方法，是ascii码为0的字符，在有些函数处理时，会把这个字符当作结束符。这是一个被广泛用于字符串处理函数的保留字符，在各种不同的业务逻辑中都可能出现问题。

文件上传漏洞中的00截断一般要在目录后面加截断0x00，%00，一般不是在文件名中加。因为程序中检测的是文件名的后缀名，如果后缀合法则拼接路径和文件名。

为什么有的放%00 有的放%00（urldecode。[]），上传的表单中有一个enctype属性，并且enctype="multipart/form-data"（不对表中数据进行编码），path大多数是存放在表单中，因此需要在数据包中进行URLdecode操作，便车个字符串结束符号。

有的path也可以存放在URL或cookie中，而在提交数据的时候浏览器会对数据做一次urldecode操作

ereg — 正则表达式匹配

ereg ( string $pattern , string $string [, array &$regs ] ) : int

以区分大小写的方式在 string 中寻找与给定的正则表达式 pattern 所匹配的子串。

%00 在数据包中以 253030 的16进制数传输，返回来的数据包中是 00 的16进制。也就是说不是_GET截断了字符串，是ereg函数。

这里ereg有两个漏洞：

  1. %00截断及遇到%00则默认为字符串的结束

  2. 当ntf为数组时它的返回值不是FALSE

这第二种nctf[]=1 ，在网站上可以实现，在本地不行，网上会报一个warning，本地报error，本地是5.6.40版本。

第一种。nctf=1%00%23biubiubiu，一定要把#写成%23，不然浏览器要把#当成锚点，不传后面的东西到服务器上。
